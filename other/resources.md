# Источники данных о книгах

## 1. Open Library API (openlibrary.org)

**Условия:** бесплатно, без ключа. Фильтр по языку: `language:rus`.  
**Документация:** openlibrary.org/developers/api  
**Минус:** международная база, доля русских книг меньше, чем у российских каталогов.

### Что можно получить

**Книга (издание / edition):**
- Идентификаторы: ISBN-10/13, OCLC, LCCN, Open Library ID (OLID), Goodreads, LibraryThing и др.
- Название, подзаголовок, описание (description).
- Авторы (имя, ссылка на карточку автора).
- Издатель, место издания, дата публикации.
- Количество страниц, вес, физический формат (Paperback и т.д.), пагинация.
- Классификация: Dewey, Library of Congress (LC).
- Тематики: subjects, места, персоны, эпохи; жанр, серия.
- Оглавление (table_of_contents), примечания (notes).
- Ссылки: страница книги, превью на archive.org (если есть полный текст), внешние ссылки.
- Обложки: URL картинок S / M / L.
- Электронные версии: список ebooks с ссылкой на archive.org.

**Произведение (work):** агрегированные данные по всем изданиям, рейтинги, полки (bookshelves).

**Автор:**
- Имя, альтернативные имена, дата рождения.
- OLID автора, список произведений (works), топ-произведение, топ-темы.

**Поиск:** книги и авторы по запросу; в документе — ключ, название, авторы, год первого издания, количество изданий, обложка, наличие полного текста, при необходимости — данные по изданиям и доступности.

**Обложки и фото:** обложки книг по ISBN/OLID/OCLC/LCCN (S/M/L); фото авторов по OLID.

## 2. Biblio Search API (bibliosearch.ru/bsapi)

**Условия:** спецификация протокола для ЭБС; реальный доступ — у конкретных библиотек/ЭБС (условия и бесплатность уточнять).  
**Формат:** JSON, до 100 результатов на запрос.

### Что можно получить

- **Название** (title), **тип документа** (ebook, journal, dissertation, hardcopy, media и др.).
- **Авторы** (authors).
- **Аннотация** (annotation).
- **Издатель** (publisher), **дата публикации** (published).
- **ISBN / ISSN.**
- **ББК, УДК** — российские библиотечные классификации.
- **ОКСО, ГРНТИ, ВАК** — коды и наименования (для учебной/научной литературы).
- **Жанр/рубрика, серия, дисциплина, ключевые слова.**
- **Обложка** (thumb), **количество страниц** (pageCount).
- **Продолжительность** (duration) — для типа media (аудио/видео).
- **Связанные документы** (related), **отрывки с совпадениями** (snippets), при поддержке — **полный текст / ссылка на полный текст**.

**Поиск:** произвольный запрос или по полям: название, авторы, ISBN, издатель, даты (publishedFrom/publishedTo), ключевые слова.

## 3. Google Books API

**Условия:** бесплатно с квотой, нужен API key.  
**Документация:** developers.google.com/books  
**Минус:** объём русскоязычной базы не гарантирован, лучше проверять по своим запросам.

### Что можно получить

- **Метаданные тома:** название, авторы, издатель, дата публикации, описание, ISBN, количество страниц, категории, язык.
- **Обложки** (thumbnail, small/medium/large).
- **Доступность просмотра** (viewability: full/partial/none), **ссылка на превью** (previewLink, infoLink).
- **Цена и наличие** (для продажи/аренды), если применимо.
- **Поиск** по запросу и метаданным, пагинация.

## 4. IPR SMART API (api.iprbooks.ru)

**Условия:** HTTPS, авторизация JWT; ориентирован на вузы и организации.  
**Назначение:** учебная и научная литература в ЭБС IPR SMART.

### Что можно получить (типично для таких API)

- Список доступных изданий (каталог).
- Метаданные изданий (название, авторы, аннотация, темы, классификации — в объёме, предусмотренном API).
- Управление пользователями и учётом доступа (в рамках контракта).
- Статистика выдачи и скачивания.

*Точный перечень полей и методов — в документации API при подключении.*
